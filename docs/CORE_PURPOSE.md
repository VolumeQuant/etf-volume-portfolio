## 1. 프로젝트의 핵심 주제

### 1.1 한 줄 정의
> **"섹터 로테이션 시그널을 자동으로 탐지하여, 투자자에게 '지금 어느 섹터로 자금이 이동하고 있는가'를 알려주는 실시간 모니터링 시스템"**

### 1.2 핵심 문제 (Problem Statement)

#### 투자자가 겪는 실제 문제
```
📊 문제 상황:
- 미국 ETF 시장에는 수천 개의 ETF가 존재
- 섹터 로테이션(자금 이동)은 매일 발생
- 하지만 개인 투자자는 "어느 섹터가 HOT한지" 실시간으로 파악 불가
- 뉴스는 사후 보도이고, 차트는 과거 데이터
- "돈이 어디로 움직이고 있는지" 실시간 시그널 부재
```

#### 기존 솔루션의 한계
| 방법 | 문제점 |
|------|--------|
| **증권사 HTS** | 개별 종목 중심, 섹터 전체 흐름 파악 어려움 |
| **금융 뉴스** | 사후 보도, 실시간 시그널 아님 |
| **ETF 스크리너** | 정적 데이터, 거래량 변화 감지 불가 |
| **블룸버그 단말기** | 비용 $24,000/년, 개인 투자자 접근 불가 |

### 1.3 우리의 솔루션 (Solution)

#### 핵심 가치 제안 (Value Proposition)
```
✅ ETF Pulse는:
1. 거래량 스파이크를 자동 탐지 (20일 이동평균 대비)
2. 11개 섹터를 히트맵으로 시각화 (HOT/COLD 한눈에 파악)
3. AI가 "왜 이 섹터에 자금이 몰리는가" 뉴스 분석
4. 실시간 알림으로 타이밍 놓치지 않게 지원

👉 결과: "지금 시장에서 무슨 일이 일어나고 있는지" 아침 3분 안에 파악
```

#### 차별점
| 기존 도구 | ETF Pulse |
|----------|-----------|
| 과거 데이터 분석 | 실시간 시그널 탐지 |
| 개별 종목 중심 | 섹터 로테이션 중심 |
| 수동 모니터링 | 자동 알림 |
| 데이터만 제공 | AI 인사이트 제공 |
| 비용 $수천~수만불 | 무료 (오픈소스) |

## 2. 이 프로젝트를 하는 진짜 이유

### 2.1 비즈니스 목적 (Primary Goal)
**"HTS 개발자 → AI/데이터 서비스 부서 직무 전환 증명"**

#### 증명해야 할 역량
```
1. 데이터 엔지니어링
   - 금융 데이터 수집 (yfinance, RSS)
   - 데이터 파이프라인 구축
   - 데이터베이스 설계 (SQLite → PostgreSQL)

2. 데이터 분석
   - 거래량 패턴 분석
   - 시계열 데이터 처리
   - 통계적 이상 탐지

3. AI/LLM 활용
   - Groq API 연동
   - 프롬프트 엔지니어링
   - RAG (뉴스 + 거래량 데이터)

4. 풀스택 개발
   - React + TypeScript 프론트엔드
   - FastAPI 백엔드
   - 실시간 통신
   - 배포 (Vercel + Railway)

5. 금융 도메인 이해
   - ETF 섹터 분류 (GICS)
   - 거래량-가격 관계
   - 섹터 로테이션 개념
```

### 2.2 사용자 가치 (Secondary Goal)
**"실제로 내가 투자에 사용할 수 있는 도구"**

#### 실사용 시나리오
```
매일 아침 9시 (미국 장 마감 직후):
1. 텔레그램 알림 확인: "XLK (Technology) EXTREME 스파이크!"
2. ETF Pulse 대시보드 접속
3. 섹터 히트맵 확인: Technology가 빨간색 (HOT)
4. AI 요약 읽기: "엔비디아 실적 발표, 반도체 섹터 상승"
5. 판단: "기술주로 자금 이동 중, QQQ 매수 타이밍"
6. 실제 거래 실행

👉 3분 안에 시장 국면 파악 → 투자 의사결정
```

### 2.3 학습 목표 (Tertiary Goal)
**"실무에서 바로 쓸 수 있는 기술 스택 마스터"**

#### 학습 우선순위
| 우선순위 | 기술 | 이유 |
|---------|------|------|
| 🔴 필수 | React + TypeScript | 증권사 표준 프론트엔드 |
| 🔴 필수 | FastAPI | 마이크로서비스 트렌드 |
| 🔴 필수 | LLM API | AI 서비스 핵심 |
| 🟡 중요 | pandas | 데이터 분석 필수 |
| 🟡 중요 | PostgreSQL | 프로덕션 DB |
| 🟢 선택 | Docker | 배포 표준 |

## 3. 프로젝트의 경계 (Scope)

### 3.1 우리가 하는 것 (In Scope) ✅

```
1. 데이터 수집
   - ETF 거래량/가격 데이터 (yfinance)
   - 금융 뉴스 (RSS/API)
   - 일일 1회 자동 수집

2. 시그널 탐지
   - 거래량 스파이크 (20일 MA 대비)
   - 이벤트 레벨 분류 (4단계)
   - 섹터별 집계

3. 시각화
   - 섹터 히트맵 (11개 섹터)
   - 거래량/가격 차트
   - 이벤트 타임라인

4. AI 인사이트
   - 뉴스 자동 요약 (LLM)
   - 거래량-뉴스 연관 분석
   - "왜 이 섹터인가" 설명

5. 알림
   - 텔레그램 실시간 알림
   - EXTREME 이벤트 자동 발송
   - 구독 관리
```

### 3.2 우리가 하지 않는 것 (Out of Scope) ❌

```
1. 투자 추천 절대 금지
   ❌ "XLK 매수하세요" → 금융 규제 위반
   ✅ "XLK 거래량 2.3배 스파이크 발생" → 사실 전달

2. 실시간 트레이딩
   ❌ 증권사 API 연동 자동 매매
   ✅ 시그널만 제공, 판단은 사용자

3. 개별 종목 분석
   ❌ AAPL, NVDA 등 종목 추천
   ✅ 섹터 ETF만 분석

4. 백테스팅 수익률 강조
   ❌ "이 전략으로 연 50% 수익"
   ✅ "과거 패턴 참고용" (교육 목적)

5. 유료 서비스
   ❌ 구독료, 프리미엄 기능
   ✅ 완전 무료 오픈소스
```

### 3.3 추후 확장 가능성 (Future Scope) 🔮

```
Phase 4 (6개월 이후):
1. 글로벌 ETF 확장 (유럽, 아시아)
2. 산업 ETF 상세 분석 (SOXX, XBI 등)
3. 포트폴리오 추적 (내 보유 ETF 모니터링)
4. 커뮤니티 기능 (사용자 인사이트 공유)
5. 모바일 앱 (React Native)
```

## 5. 프로젝트의 핵심 가정 (Key Assumptions)

### 5.1 기술 가정
```
1. yfinance API가 계속 작동한다
   - 리스크: Yahoo Finance 정책 변경
   - 대응: Alpha Vantage 백업 준비

2. Groq API가 무료/저렴하게 유지된다
   - 리스크: 가격 인상
   - 대응: Rule-based 폴백

3. React 기술이 2025년에도 표준이다
   - 리스크: 신기술 등장
   - 대응: 컴포넌트 설계로 마이그레이션 용이
```

### 5.2 도메인 가정
```
1. 거래량 스파이크 = 시장 관심도 증가
   - 검증: 백테스팅으로 상관관계 확인

2. 섹터 로테이션은 실제로 존재한다
   - 검증: 역사적 데이터 분석

3. 뉴스와 거래량은 연관성이 있다
   - 검증: 이벤트 전후 뉴스 키워드 분석
```

### 5.3 사용자 가정
```
1. 투자자는 "왜"를 알고 싶어한다
   - 검증: AI 인사이트 사용률 측정

2. 아침 3분이면 충분하다
   - 검증: 실제 사용 시간 측정

3. 텔레그램 알림이 유용하다
   - 검증: 알림 구독률 측정
```

## 6. 프로젝트 철학 (Design Philosophy)

### 6.1 핵심 원칙

#### 1. 단순함 우선 (Simplicity First)
```
❌ 잘못된 접근:
- 모든 ETF 분석 (수천 개)
- 복잡한 ML 모델
- 과도한 기능

✅ 올바른 접근:
- 11개 섹터 ETF만 (SPDR Select)
- 단순한 통계 (이동평균)
- 핵심 기능만 (히트맵, 알림)

이유: 6개월 안에 완성 + 실사용 가능
```

#### 2. 데이터 중심 (Data-Driven)
```
❌ 감으로 판단: "기술주가 오를 것 같아"
✅ 데이터로 판단: "XLK 거래량 2.3배 스파이크"

모든 판단은 수치 기반:
- 거래량 비율 (정량)
- 뉴스 빈도 (정량)
- 가격 변화 (정량)
```

#### 3. 규제 준수 (Compliance)
```
투자 추천 금지:
❌ "지금 매수하세요"
❌ "이 ETF가 오를 것입니다"
❌ "수익률 보장"

✅ "데이터 전달만":
- "거래량 스파이크 발생"
- "관련 뉴스: 실적 발표"
- "과거 패턴: 참고용"

👉 모든 판단은 사용자 책임
```

#### 4. 실사용 가능 (Production-Ready)
```
데모용이 아닌 실제 사용:
- 매일 아침 9시 자동 실행
- 안정적 에러 처리
- 빠른 로딩 속도 (<3초)
- 모바일 반응형

👉 본인이 30일간 실제 사용
```

## 8. 핵심 질문과 답변 (FAQ)

### Q1. 왜 ETF인가? 주식은 안 되나?
```
A: 개별 주식은 수천 개, 분석 불가능.
   ETF는 섹터 대표성 + 높은 유동성
   👉 섹터 로테이션 분석에 최적
```

### Q2. 왜 거래량인가? 가격은?
```
A: 거래량 = 자금 이동의 실제 신호
   가격은 이미 반영된 과거 정보
   👉 거래량이 가격보다 선행
```

### Q3. 왜 AI가 필요한가?
```
A: 거래량만으로는 "왜"를 모름
   뉴스 분석으로 원인 파악
   👉 투자자에게 맥락 제공
```

### Q4. 이게 실제로 돈이 되나?
```
A: 직접 목적은 "직무 전환"
   하지만 실사용 가능해야 설득력
   👉 본인이 30일 실제 사용 예정
```

### Q5. 왜 오픈소스인가?
```
A: 포트폴리오 목적 + 기술 증명
   유료화는 규제 리스크
   👉 GitHub로 기술 역량 증명
```
